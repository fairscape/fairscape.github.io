    Introduction

    FAIRSCAPE is a reusable computational framework, enabling simplified access to modern scalable cloud-based components. It fully implements the FAIR data principles and extends them to provide FAIR Evidence, including provenance of datasets, software and computations,as metadata for all computed results.

    Authentication

    To authorize, use this code:

    import FAIR

token = 'your-JWT'

    # With shell, you can just pass the correct header with each request
curl "api_endpoint_here" \
  -H "Authorization: JWT"

    Make sure to replace `JWT with your web token.

    FAIRSCAPE uses json web tokens to authenticate users. Please our LOGIN to receive your web token.

    FAIRSACPE expects the web token to be included in all API requests to the server in a header that looks like the following:

    Authorization: your-web-token

    You must replace your-web-token with your personal json web token.

    MDS

    Endpoints

    • /ark:{prefix}
    • /shoulder/ark:{namespace}
    • /ark:{namespace}/{Identifier}

    /ark:{prefix} Namespace Handling

    Handles creating and editing namespaces to store identifiers. {prefix} is 5 digit ark namespace prefix

    /shoulder/ark:{namespace}

    Handles Identifier Minting

    /ark:{namespace}/{Identifier}

    Identifier Handling Resolves, updates and deletes identifiers.

    /mds/ark:{prefix} GET

    upload_response = requests.get(
        'http://clarklab.uvarc.io/mds/ark:{prefix}',
        headers = {"Authorization": token}
    )

    $ curl http://clarklab.uvarc.io/ark:99999

    The above returns the json-ld metadata of the given namespace. May look like below:

    {
  "name":"Test",
  "description":"Example Namespace"
}

    Get namespace description. Returns metadata describing requested namespace.

    /mds/ark:{prefix} POST

    $ curl --request POST \
  --url https://clarklab.uvarc.io/mds/ark:99999 \
  --header 'Authorization: Bearer YOUR_JWT' \
  --header 'Content-Type: application/json' \
  --data '{"name":"Test","description":"Test Namespace"}'

    metadata = {
    'name':'Example Namespace',
    'description':'Sample'
    }
ns_response = requests.post(
        'http://clarklab.uvarc.io/mds/ark:99999',
        data = json.dumps(metadata),
        headers = {"Authorization": token}
    )

    Create a namespace. Prefix is a 5 digit number representing your institution.

    Parameters

    Parameter Required Description
    name false Name of object.
    description false Description of object

    Transfer

    Takes data from the user and uploads the given file to minio and mints an identifier in mongo associated with the file.

    Endpoints

    /data

    Handles uploading of new data.

    /data/{PID}

    Endpoint for download, updating, or deleting previously uploaded file.

    /transfer/data POST

    $ curl --request POST \
  --url https://clarklab.uvarc.io/data \
  --header 'Authorization: Bearer YOUR_JWT' \
  --header 'Content-Type: application/json' \
  --data '{metadata:{"name":"Example Dataset", "@type":"Datatset", "description":"Example made up data"},
  "files":@myfile,
  "sha256":"sampple-sha"
}'

    metadata = {
    'name':'Example Dataset',
    '@type':'Dataset',
    'author':'Madeup Person',
    'description':'Sample'
    }
upload_response = requests.post(
        'http://clarklab.uvarc.io/transfer/data',
        files = {
            'files':open(file_path,'rb'),
            'metadata':json.dumps(metadata),
            'sha-256':'abcdefghi'
        },
        headers = {"Authorization": token}
    )

    The above returns json like below:

    {
  "upload":"True",
  "Minted Identifier":["ark:99999/abd-123"]

}

    The user passes in a file to upload as well as json-ld metadata. The service uploads the file to minio and mints an identifier with the given metadata as well as a distribution tag tying the file to the identifier. The service returns the minted identifier to the user.

    Parameters

    • Files
      • Metadata
      • Files
      • sha-256

    /transfer/data/{PID} GET

    $ curl http://clarklab.uvarc.io/data/{pid}

    upload_response = requests.get(
        'http://clarklab.uvarc.io/data/{pid}',
        headers = {"Authorization": token}
    )

    Commands above return the file in content.

    Download file associated with given PID.

    /transfer/data/{PID} PUT

    $ curl --request PUT \
  --url https://clarklab.uvarc.io/data/{PID} \
  --header 'Authorization: Bearer YOUR_JWT' \
  --header 'Content-Type: application/json' \
  --data '{metadata:{"name":"Example Dataset", "@type":"Datatset", "description":"Example made up data"},
  "files":@myfile,
  "sha256":"sampple-sha"
}'

    update_response = requests.put(
        'http://clarklab.uvarc.io/transfer/data/{pid}',
        files = {
            'files':open(file_path,'rb'),
            'sha-256':'abcdehgia'
        },
        headers = {"Authorization": token}
    )

    The above returns json like below:

    {
  "upload":"True",
  "Minted Identifier":["ark:99999/abd-123"]

}

    This endpoint is used to upload a new version of a previously uploaded file. The user passes in a file to upload. The service uploads the file to minio and mints a new distribution tag tying the file to the identifier. The service returns the minted identifier to the user.

    Parameters

    • Files
      • Files (can be list or single file)
      • sha-256

    /transfer/data/{PID} DELETE

    $ curl --request DELETE
       --url http://clarklab.uvarc.io/data/{pid}

    delete_response = requests.delete(
        'http://clarklab.uvarc.io/data/{pid}',
        headers = {"Authorization": token}
    )

    Delete a file associated with given PID from FAIRSCAPE.

    Compute

    The compute service handles creating and tracking of user uploaded jobs. The service has 3 options: a custom nypipe container with detailed provenance tracking, spark on kubernetes, and option to input the container of your choice.

    Endpoints

    • /job
    • /nypipe
    • /spark

    /job

    Handles tracking running jobs and creating jobs with arbitrary containers.

    /nypipe

    Handles creating jobs with custom nypipe container for tracking detailed provenance.

    /spark

    Handles creating spark jobs.

    /compute/job GET

    The command below returns all running jobs:

    $ curl http://clarklab.uvarc.io/compute/job

    jobs = requests.get(
        'http://clarklab.uvarc.io/compute/job',
        headers = {"Authorization": token}
    )

    The above returns json like below:

    {
  "jobs":["nypipe-99999/abd-123","spark-9999/tests-job"]

}

    List all running jobs.

    /compute/job POST

    $ curl --request POST \
  --url https://clarklab.uvarc.io/job \
  --header 'Authorization: Bearer YOUR_JWT' \
  --header 'Content-Type: application/json' \
  --data '{"datasetID":"ark:99999/data", "scriptID": "ark:99999/script", "containerID":"ark:99999/dockerID"}'

    job_response = requests.post(
        'http://clarklab.uvarc.io/compute/job',
        data = json.dumps({
            'scriptID':"ark:99999/script",
            "datasetID":"ark:99999/data",
            "containerID":"ark:99999/dockerID"
        }),
        headers = {"Authorization": token}
    )

    Starts a new job computing the given script on the requested data. Returns identifier of running job.

    Parameters

    Parameter Required Description
    datasetID True List of IDs of data to be computed on.
    scriptID True ID of script to run on given data.
    containerID True ID of container in which to run the computation

    /compute/nypipe POST

    $ curl --request POST \
  --url https://clarklab.uvarc.io/nypipe \
  --header 'Authorization: Bearer YOUR_JWT' \
  --header 'Content-Type: application/json' \
  --data '{"datasetID":"ark:99999/data", "scriptID": "ark:99999/script"}'

    job_response = requests.post(
        'http://clarklab.uvarc.io/compute/nypipe',
        data = json.dumps({
            'scriptID':"ark:99999/script",
            "datasetID":"ark:99999/data"
        }),
        headers = {"Authorization": token}
    )

    Starts a new job computing the given script on the requested data. Returns identifier of running job.

    Parameters

    Parameter Required Description
    datasetID True List of IDs of data to be computed on.
    scriptID True ID of script to run on given data.

    /compute/spark POST

    $ curl --request POST \
  --url https://clarklab.uvarc.io/spark \
  --header 'Authorization: Bearer YOUR_JWT' \
  --header 'Content-Type: application/json' \
  --data '{"datasetID":"ark:99999/data", "scriptID": "ark:99999/script"}'

    job_response = requests.post(
        'http://clarklab.uvarc.io/compute/spark',

    Starts a new job computing the given script on the requested data. This endpoint utlizes spark on kubernetes to allow for processing of large data. Returns identifier of running job.

    Parameters

    Parameter Required Description
    datasetID True List of IDs of data to be computed on.
    scriptID True ID of script to run on given data.

    Evidence Graph

    The evidence graph service creates a detailed provenance trace of the requested identifier. The returned evidence graph is a json-ld representation of the computations and inputs that went into creating the requested object.

    /eg/{PID} GET

    jobs = requests.get(
        'http://clarklab.uvarc.io/eg/{PID}',
        headers = {"Authorization": token}
    )

    $ curl http://clarklab.uvarc.io/eg/{PID}

    Returns evidence graph for requested PID

    Visualization

    The visualization service takes the evidence graph of a PID and displays an html graph of the evidence graph.

    /viz/{PID} GET

    jobs = requests.get(
        'http://clarklab.uvarc.io/viz/{PID}',
        headers = {"Authorization": token}
    )

    $ curl http://clarklab.uvarc.io/ark:99999/viz/{PID}

    Visualizes evidence graph of requested PID using cytoscape.

    Search

    The search service runs string match searches over the metadata store and returns matches in order by search fit.

    /search/{query} GET

    Search metadata for identifiers containing test

    
search_request = requests.get("http:/clarklab.uvarc.io/search/test")

    $ curl http://clarklab.uvarc.io/search/test

    The above returns json like below:

    {
  "matches":[
    "ark:99999:abc-231",
    "ark:99999:abfjks-343hgh",
    "ark:99999:ver-2343"
  ]
}

    Searches the metadata store for literals matching the requested query. Search is based off Lucene 7.4 and supports all the same modifiers as Lucene .

    Errors

    The FAIRSCAPE API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The service requested is for administrators only.
    404 Not Found.
    405 Method Not Allowed.
    406 Not Acceptable -- You requested a format that isn't json.
    418 I'm a teapot.
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
